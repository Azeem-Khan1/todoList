Todo List Project

Welcome to the Todo List project presentation. In this project, we will create a simple and interactive todo list application using HTML, CSS, and JavaScript.

Project Overview

  • Create a todo list interface
  • Add functionality to add new todos
  • Allow editing and deleting todos
  • Implement marking todos as completed
  • Persist the todo list using local storage

In-Depth JavaScript Breakdown

  1. Get references to the necessary HTML elements:
    • todoInput: Represents the input field where the user enters new todos.
    • addBtn: Represents the button to add a new todo.
    • todoList: Represents the unordered list (UL) where the todos will be displayed.
  2. Enable the Enter key to add a new todo:
    • Attach an event listener to the todoInput element for the keyup event.
    • When the Enter key (key code 13) is pressed, trigger the addBtn.click() function.
  3. Initialize the todos array:
    • Create an empty array to store the todos.
  4. Check local storage for existing todos:
    • If there are todos stored in the local storage, retrieve and parse them into the todos array.
    • Call the renderTodos() function to display the todos on the page.
  5. Add a new todo:
    • Attach a click event listener to the addBtn button.
    • When the button is clicked, perform the following steps:
      • Retrieve the trimmed value from the todoInput field.
      • If the input is not empty, create a new todo object with a unique ID, the input text, and a completion status of false.
      • Add the new todo to the todos array.
      • Save the updated todos array to local storage.
      • Call the renderTodos() function to update the displayed todos on the page.
      • Reset the todoInput value to an empty string.
  6. Edit a todo:
    • Attach a click event listener to the todoList element.
    • When a click event occurs on an element with the class todo-text, perform the following steps:
      • Retrieve the ID of the parent element (list item) from the data-id attribute.
      • Find the corresponding todo in the todos array.
      • Prompt the user to enter the updated text for the todo.
      • If the user provides a new text (not null), update the todo's text property and save the changes to local storage.
      • Call the renderTodos() function to update the displayed todos on the page.
  7. Delete a todo:
    • Attach a click event listener to the todoList element.
    • When a click event occurs on an element with the class delete-btn, perform the following steps:
      • Retrieve the ID of the parent element (list item) from the data-id attribute.
      • Filter out the todo with the matching ID from the todos array.
      • Save the updated todos array to local storage.
      • Call the renderTodos() function to update the displayed todos on the page.
  8. Complete a todo:
    • Attach a click event listener to the todoList element.
    • When a click event occurs on an element with the class complete-btn, perform the following steps:
      • Retrieve the ID of the parent element (list item) from the data-id attribute.
      • Find the corresponding todo in the todos array.
      • Toggle the completion status of the todo (true to false, false to true).
      • Save the updated todos array to local storage.
      • Call the renderTodos() function to update the displayed todos on the page.
  9. Render todos:
    • Clear the contents of the todoList element.
    • Iterate over each todo in the todos array and create the necessary HTML elements to display them.
    • Apply appropriate CSS styling.
